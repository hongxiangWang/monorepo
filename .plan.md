## 核心配置文件

### pnpm-workspace.yaml

```yaml
packages:
  - 'packages/*'
  - 'apps/*'
  - '!**/node_modules'
```

### 根目录 package.json

```json
{
  "name": "monorepo-root",
  "private": true,
  "packageManager": "pnpm@8.x",
  "workspaces": ["packages/*", "apps/*"],
  "scripts": {
    "build": "turbo build",
    "dev": "turbo dev",
    "lint": "turbo lint",
    "type-check": "turbo type-check",
    "clean": "rimraf --glob '*/**/dist' '*/**/node_modules'",
    "format": "prettier --write \"**/*.{ts,tsx,md,json}\"",
    "test": "turbo test"
  },
  "devDependencies": {
    "prettier": "^3.0.0",
    "rimraf": "^5.0.0",
    "turbo": "^1.10.0"
  }
}
```

## 主要特点

### 为什么用 pnpm 的 monorepo：

- 🚀 **链接依赖**：pnpm 使用硬链接和符号链接，节省磁盘空间，安装速度快
- 📦 **原生支持**：pnpm 原生支持 monorepo，不需要额外工具（如 lerna）
- ⚡ **高效缓存**：Turborepo 提供远程缓存，加速构建

## 开发工作流

### 常用命令：

```bash
# 安装所有依赖
pnpm install

# 在所有包中运行构建
pnpm build

# 同时开发多个应用
pnpm dev

# 只运行特定应用的开发
pnpm --filter web dev

# 在特定包中安装依赖
pnpm add lodash --filter utils

# 使用工作区内的包
# 在 app/web/package.json 中声明: "utils": "workspace:*"
```

## 包间依赖管理

当某个应用需要使用工作区内的共享包，只需在对应的 `package.json` 中添加：

```json
{
  "dependencies": {
    "@repo/ui": "workspace:*",
    "@repo/utils": "workspace:*"
  }
}
```

`workspace:*` 表示始终使用工作区内的最新版本。

## 版本发布策略

- **独立版本**：每个包独立版本号，使用 changesets 管理
- **统一版本**：所有包使用同一个版本号，适合紧密关联的项目

推荐使用 [changesets](https://github.com/changesets/changesets) 管理版本和发布。
